# EVALUATION: Performance & Scalability Benchmarking

This evaluator details the requirements to certify that the storefront and API endpoints load within required sub-second metrics.

---

## ⚡ Criteria Checklist

### 1. Google Lighthouse Audits
Lighthouse evaluations run against storefront routes (Homepage, Category grid, Product Detail, Sizing Wizard, and Checkout) in Staging must meet the following thresholds:
- [ ] **Performance**: >= 90
- [ ] **Accessibility**: >= 95
- [ ] **Best Practices**: 100
- [ ] **SEO**: 100
- [ ] First Contentful Paint (FCP) must be less than **1.0 second**.
- [ ] Largest Contentful Paint (LCP) must be less than **2.0 seconds**.

### 2. API Response Latency
- [ ] Public catalog endpoints (`GET /api/v1/products`) must remain under **500ms** at p95 under load.
- [ ] Warm vs. cold catalog load times must demonstrate a cache hits latency reduction of at least **200ms**.

### 3. Caching & Caching Indexes
- [ ] Active Redis cache layers (TTL 60s) must store responses for frequently called catalog endpoints.
- [ ] Databases must perform index scans for `EXPLAIN ANALYZE` on search and catalog queries, avoiding heavy sequential scans.

### 4. Media & Code Optimization
- [ ] Product image assets must utilize Next.js `<Image>` mapping with `placeholder="blur"` and convert to WebP/AVIF.
- [ ] Web bundles generated by Next.js build scripts must not exceed **200KB** (gzipped) for individual files.
- [ ] Silent hero background videos must be served in WebM (with MP4 fallbacks) and load only after first user interaction or after a 3s delay to prevent LCP blocking.

### 5. Concurrent Load Testing (k6 / Locust)
- [ ] Catalog browsing routes must handle **500 concurrent users** at p95 < 500ms response times.
- [ ] Checkout webhooks processing must resolve in less than **2.0 seconds** for 50 concurrent completions.
